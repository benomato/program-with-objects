Stage 1: Make your webpage

Lesson 1: The Basics of the web and HTML

Some context

World Wide Web: a collection of HTML documents.

HTML: hypertext Markup Language.

Hyperlinks: what makes the web web like.

What type of files can be found on the web?

  • Plain text
  • HTML
  • Images
  • Videos
  • Music

Major pieces

The browser makes requests via internet to servers. These requests are using a protocol called HTTP and the servers respond with files that the browser displays.

HTML

HTML is made of:

  • text content: What I see
  • markup: What it looks like / how it is arranged
  • references to other documents: e.g: images, videos.
  • links to other pages

Plain text is plain text in HTML.

HTML Markups

HTML is made of things called tags.

We can write it like this: <name>contents</name>

list of the principal tags:

  • <b> -----> bold
  • <em> -----> emphasis, italics
  • <a href="....."> -----> anchor, anchors are made for links
  • <img src="url" alt="text"> -----> images
  • <br> -----> break, inline tag, go to next line
  • <p> -----> paragraph, block tag, makes an invisible box
  • <span> -----> normal tag, inline
  • <div> -----> normal tag, block
  • <!DOCTYPE html> -----> Doctype to use at the beginning of every html file.
  • <html> -----> opening html tag
  • <head> -----> metadata and random stuff
  • <title> -----> Title of the page, see it in tab
  • <body> -----> The actual content of the HTML document

Lesson 2: Creating a structured document

A website is like a house:

  • HTML files are like the structure.
  • CSS files are then the style of your house.
  • JS files are the interactive components(e.g: television remote).
  • It also exists some rules to follow(e.g: DOM).
  • Eventually, some tools can help us build our websites(e.g: Chrome developer tools).

The DOM(Document object model) is the tree like structure of an html page that a browser interprets.

Everything in a webpage is a box.
Boxify a design means break down it in several boxes.
To help create this box in the html, we can use the div tag.
To bring style to this "div", we add the attribute class that allows to use CSS.

Lesson 3: Adding CSS Style to HTML structure

Welcome to CSS

CSS is a language that avoids programmers to repeat things over and over. Using html tags and other selectors, you can quickly add the same style to different sections of your HTML code. It saves a lot of time and lines of code.

In the CSS acronym, C stands for cascading. It is a really interesting rules that means that the most specific CSS rule applies to an HTML element.

To style a page using CSS, you need to choose a selector and apply some CSS properties. For example, if you want to change the color of an h2 to red you do:

h2  {
color: red;
}

Everything in a webpage is a box

A web page could be described as a box model: it is a box, that has a box that has a box etc..

The box model includes, from outside in: margin, border, padding, content. We can find a great explanation here: W3schools

The "True" width of an HTML element is the sum of content + padding + border. The box-sizing CSS property specifies that elements should have padding and border included in the element's total width and height. It doesn't include margins.

Display blocks in the same line

CSS property display sets to value flex allows to display block elements on the same line. For some HTML block elements in a container, CSS property display must be applied to the container so the block elements could be on the same line.

Code, test and refine

One you received the design mockup, you can follow these steps:

  1. Look for the natural boxes and try to boxify the design
  2. Look for styles that repeat
  3. Write the HTML
  4. Apply styles (from the biggest to the smallest)
  5. Test and fix
  6. Repeat steps 3 to 5 until your webpage matches the design

Lesson 2: Variables and strings

Variables

A variable is a container where we can store values such as numbers, strings etc. It is a way to give names to values.

To set a variable, we write an assignment statement:

name  =  Expression

Variables can vary. We can change the value of a variable by re-assigning it to a different value later.

We also need to define a variable prior to make any calculation with it.

Strings

A string is just a sequence of characters surrounded by quotes. It can be single or double quotes.

Concatenation

string + string

We cannot concatenate strings and numbers. However we can multiply strings.

Indexing strings

string[expression]

Expression is a number and indexing starts at position 0. It is the first letter of the word.

Selecting sub-sequence

string[start:stop]

The result is a string that is a sub-sequence of the characters in the string starting from position start and ending with position stop - 1.

Finding strings in strings

string.find(string)

The result is the number that gives first position in search string where the target string appears. If target string is not found, result is -1.

find is called a method and find is case sensitive.

string.find(string, number)

The result is a number that gives first position in search string where the target string appears at or after number. If target string is not found, result is -1.

Lesson 3: Input, function, output

What is a funtion?

A function or procedure is a piece of code that takes inputs, do things with those inputs and eventually returns outputs. It is really practical as it avoids repetition and facilitate code organization.

Making/using a function

We can make a function as follows:

def function_name(input1, input 2, ...):
    "block of code"
    return outputs

The return stament is fundamental to do things to input. Otherwise the function does nothing but printing something at best.

Using a function means giving her some inputs so she can execute code and returns the corresponding outputs.

Lesson 4: Control flow and loops: if and while

Making decisions

Making decisons in programming consists to evaluate if a statement is True or False.

IF

if test expression:
    block1
else:
    block2

If the test expression evaluates to True, it executes block1, otherwise it executes block2

OR

expression or expression

If the first expression evaluates to True, the value is True and the second expression is not evaluated

If the value of the first expression is false, the value is the value of the second expression

Loops

While

while test expression:
    block

While continues to execute the block code while the test expression is True. It stops when it is False.

Break

The keyword break allows to stop the loop even though the test expression evaluates to True.

Lesson 6: Lists and for loops

Lists

Like a string, a list is a structured data. It is a sequence of anything. To create a list, we use square brackets and elements are separated by commas.

e.g: my_list = ['a', 'b', 'c']

Mutation

List supports mutation. It means that we can change the value of a list after we created it. Modifying an element of a list will not create a new list.

Aliasing

When we have 2 different ways to refer to the same object, that is called aliasing. Therefore, if p and q refers to the same list and if you modify an element of p, it will also change it in q.

Lists can take several methods. Look for it in python documentation.

For loops

We write for loops like this:

for name in list:
    block

A for loopmeans that for each element in the list, we're going to assign that element to the name and evaluate the block. Once the loop is finished, it goes to another statement. We can break a loop using the keyword break.

Computational problem solving methodology

  1. Don't panic

  6. Develop the solution incrementally and test as you go

Stage 3: Program with objects

Lesson 1: Abstraction

Abstraction

Abstraction is the hiding of detail in programming. This is really powerful because it allows to focus on the program we write. We can then use functions simply by reading their documentation.

Like a CSS class provides a way to style elements that share similar characteristics, Objected Oriented Programming is a programming paradigm that allows to regroup data and behaviors in the same place. We can then build a general blueprint and specify objects as we need. To build this general blueprint, we use classes.

Brief class introduction

A module is a file that contains functions and variables that we can import in other modules. The defining characteristic of a module is information hiding.

A library is a collection of related functionality, whereas a module only provides a single piece of functionality. So a library will tipically contain multiple modules.

Python has its own built-in modules that compose the Python Standard Library. We can also use third party modules or create and import our own modules.

When we write python modules, what we are doing is writing code one line at a time, almost like a "recipe".

For example, using what we know about functions, if we want to build a movie website, we would create a movie.py file and we would add a bunch of functions such as show_strailer() or show_info(). So far so good.

This process is good while you manage little information. But assuming we want to get information about a movie, we would have to pass a lot of arguments in our show_info function. And that is starting to be less manageable and less smart.

In this case, what we really want to do is to define a template for our movie which is a python module and record all of the data that needs to go into that template(like the movie's title, functions etc).

In this case you can call this module movie after movie. No arguments necessary. To do that, we just have to create copies of our template to use it for the movie we want. Unfortunately, we can have several issues with this method. For example if we want to add more pieces to our code or modify a function, we will then have to do it for all of the template copies! Not that smart..

What we really need is a way of making copies of a template, without having multiple files. The goal is to take a movie, and say "hey! This movie is a type of that template". This new thing in programming is called a class. A class allows to manage bigger projects easier than using functions and modules.

Lesson 2: Use classes

Module, class and __init__

In this subsection, we consider a module which owns a class which has a __init__inside her.

__init__ is a constructor that is being called automatically when we instantiate a class and it creates space and memory for the new instance/object being accessed. In the course example, we create a class named turtle and we instantiate this class in a variable named Brad: brad = turtle.Turtle()

Brad can now access all of the rest of the methods that are inside class Turtle.

Class: A class is like the "blueprint of a building". The blueprint as we know contains certain pieces of information about any building that we are able to construct.

Instance or object: We know know that a blueprint can be used to build different buildings (e.g: blue or yellow walls). So, an instance is a specific building that is built using the general class blueprint.

Lesson 3: Make classes

To create a class, we use the keyword class and name the class with the first letter in capital.

It is good practice to keep the class definition on one file and to call your class from another file.

__init__ initializes a class. It creates space and memory. We call it a constructor.

self is the object being created and it is the first parameter in every method (function) in a class.

Instance variable: a variable that is unique to each instance of a class.

Instance method: a fonction inside a class and associated with an instance.

Lesson 4: Advanced topics

Pre-existing class variables

__doc__: it allows to acess the class documentation of a class. You write class documentation between triple-quotes.

__name__: It gives the name of the class.

__module__: It gives the name of the module where the class is.

Inheritance

Like a child can inherit traits from a parent, a Class can be a child of a Parent class and inherit traits (data and methods) from him.

We can define inheritance as the concept that a class can inherit traits from another class.

A Child class can implicitly use a Parent class method, override it or call a Parent class method even though it has overriden it.
